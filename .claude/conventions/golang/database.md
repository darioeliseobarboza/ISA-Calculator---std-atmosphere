# Database (Go, sqlc + pgx)

SQL-first data access: queries are written in `.sql` files and [sqlc](https://sqlc.dev) generates type-safe Go from them, executed over [pgx](https://github.com/jackc/pgx) (PostgreSQL). No ORM, no reflection, no query magic. Each domain module exposes a repository interface; the generated code is an implementation detail behind it.

## When to use

Services that own a relational database (PostgreSQL by default). Services whose state lives in a key-value/document store or a message bus use a different adapter — see `## Non-SQL variant`.

## Package

```
github.com/sqlc-dev/sqlc        # codegen (dev tool, pinned)
github.com/jackc/pgx/v5         # driver + pool
github.com/jackc/pgx/v5/pgxpool # connection pool
```

Migrations: `github.com/golang-migrate/migrate` (or `goose`), SQL files versioned in `db/migrations/`.

## Structure

```
db/
├── migrations/                # NNNN_description.up.sql / .down.sql
├── queries/                   # *.sql with -- name: annotations for sqlc
└── sqlc.yaml                  # sqlc config (engine: postgresql, output package)
internal/
└── order/
    ├── order.go               # domain type
    ├── repository.go          # Repository interface (the port)
    └── repository_pg.go       # adapter: wraps the sqlc-generated queries
```

## How to use

### Define the port where it is consumed

```go
// internal/order/repository.go
type Repository interface {
	Create(ctx context.Context, o Order) (Order, error)
	GetByID(ctx context.Context, id string) (Order, error)
}
```

### Write the query (sqlc generates the Go)

```sql
-- db/queries/order.sql
-- name: CreateOrder :one
INSERT INTO orders (id, customer_id, total, currency, status)
VALUES ($1, $2, $3, $4, $5)
RETURNING *;
```

### Implement the adapter over the generated code

```go
// internal/order/repository_pg.go
type pgRepository struct {
	pool *pgxpool.Pool
	q    *db.Queries // generated by sqlc
}

func NewPGRepository(pool *pgxpool.Pool) Repository {
	return &pgRepository{pool: pool, q: db.New(pool)}
}

func (r *pgRepository) GetByID(ctx context.Context, id string) (Order, error) {
	row, err := r.q.GetOrder(ctx, id)
	if err != nil {
		if errors.Is(err, pgx.ErrNoRows) {
			return Order{}, ErrNotFound
		}
		return Order{}, fmt.Errorf("get order %s: %w", id, err)
	}
	return toDomain(row), nil
}
```

The repository maps driver errors to **domain errors** (`pgx.ErrNoRows` → `ErrNotFound`) and DB rows to **domain types** (`toDomain`). The service never sees `pgx` or generated types.

### Transactions

Pass a transaction-scoped `Queries` when several writes must be atomic:

```go
func (r *pgRepository) WithTx(ctx context.Context, fn func(Repository) error) error {
	tx, err := r.pool.Begin(ctx)
	if err != nil {
		return err
	}
	defer tx.Rollback(ctx) //nolint:errcheck // no-op after commit
	if err := fn(&pgRepository{q: db.New(tx)}); err != nil {
		return err
	}
	return tx.Commit(ctx)
}
```

## Migrations

- Forward-only in production; every migration has an `.up.sql` and a `.down.sql` for local rollback.
- Named `NNNN_short_description.up.sql` (zero-padded sequence).
- Run on deploy (separate step), never implicitly at app startup in production.

## Rules

- Queries live in `.sql` files; Go DB code is **generated** by sqlc. Do not hand-write SQL string concatenation in Go.
- One repository interface per module, defined in the module package (the port). The pg adapter is unexported behind a constructor.
- The repository maps driver errors to domain errors and rows to domain types. `pgx`/generated types never leak past the repository.
- Always use a connection **pool** (`pgxpool`), created once and injected. Never open per-request connections.
- Every query takes `ctx` and respects cancellation/timeouts.
- Schema changes go through versioned migrations. No manual DDL in production.
- No `SELECT *` in code paths where columns matter for stability; prefer explicit columns (the `RETURNING *` in inserts is acceptable since sqlc types it).

## Non-SQL variant

When the service's state lives in a key-value/document store, a cache, or a message bus (e.g., Redis, JetStream KV), the **port stays the same** (a `Repository` interface in the module package) and only the adapter changes (`repository_kv.go`). The rules about mapping store errors to domain errors, mapping records to domain types, injecting the client, and passing `ctx` all still apply. Document the chosen store in `overview.md` and, if it replaces this catalog default, as a custom convention.

## Integration with other conventions

- **error-handling**: the repository translates store errors to domain/typed errors.
- **observability**: wrap the pool/queries with tracing instrumentation when active.
- **config**: the connection string comes from `Config` (`DATABASE_URL`).
