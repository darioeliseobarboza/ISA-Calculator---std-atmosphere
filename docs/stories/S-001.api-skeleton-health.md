# S-001: Esqueleto atmosphere-api + GET /health

**Status:** Completed
**Created:** 2026-06-15
**Source:** REQ-001

## Historia de Usuario

**Como** usuario técnico / operador del sistema,
**quiero** un servicio `atmosphere-api` (Go) que exponga `GET /health`,
**para** confirmar que el backend está operativo (prueba de vida del esqueleto).

## Criterios de Aceptación

### CA-1: Health responde estado saludable

**DADO** `atmosphere-api` corriendo
**CUANDO** consulto `GET /health`
**ENTONCES** responde `200` con cuerpo JSON `{ "status": "ok", "timestamp": <RFC3339> }`

### CA-2: CORS habilitado y configurable

**DADO** la API configurada con `CORS_ALLOWED_ORIGINS`
**CUANDO** un origen permitido hace una request cross-origin a `/health`
**ENTONCES** la respuesta incluye los headers CORS correspondientes (sin error de CORS)

### CA-3: Configuración por entorno

**DADO** las variables `HTTP_ADDR` y `CORS_ALLOWED_ORIGINS`
**CUANDO** arranca el servicio
**ENTONCES** escucha en el address configurado y aplica los orígenes CORS; si falta una variable requerida, falla al arrancar (fail-fast)

### Cambios de API

#### Endpoints Existentes a Modificar
Ninguno.

#### Nuevos Endpoints

**`GET /health`** (ya especificado en `docs/apis/atmosphere-api.yaml`, `operationId: healthCheck`)
- **Servicio:** atmosphere-api
- **Descripción:** Health check / liveness, sin auth.
- **Response 200:**
  ```json
  {
    "status": "string — \"ok\"",
    "timestamp": "string — RFC3339 UTC"
  }
  ```
- **Errores:** ninguno esperado (endpoint barato).

> `POST /v1/calculate` queda **fuera** de esta story (FG-2/3).

### Cambios de Base de Datos
Ninguno (servicio stateless, ADR-003).

### Flujo de Interacción

#### Escenario: Liveness check (lado API)

**Trigger:** un cliente (la app o curl) consulta `GET /health`.

**Flujo:**
1. **[atmosphere-api]** middleware CORS evalúa el origen; el handler de `/health` responde.
2. **[atmosphere-api]** devuelve `200 { "status": "ok", "timestamp": ... }`.

**Outcome:** el backend confirma que está vivo.

**Manejo de errores:**
- Falta de config requerida al arranque → el proceso no levanta (fail-fast, ver `config`).

### Flujos Afectados

- Ninguno documentado se modifica. La interacción `health-check` es trivial (no requiere doc de flujo propio); el flujo principal sigue siendo `calculate` (FG-2/3).

### Consideraciones Técnicas

**Seguridad:** sin auth (ADR-001); CORS configurable por env. TLS lo termina el proxy en prod.
**Performance:** `/health` es barato (liveness).
**Dependencias:** ninguna (primera story); habilita S-002 y S-003.
